## 3. 结论摘要
1. DSO midpoint 偏负主要是当前 DER 组合造成的，不是简单由基础负荷缺失造成。EVCS、柔性负荷、HVAC 是天然吸收型资源；当这些资源在 VPP 中占比较高时，聚合可行域的中点会向负功率方向偏移。
2. 当前组合可以作为“配电网灵活性压力测试”使用，但如果论文要论证 VPP 利润最大化，它还不够真实：EV 用户充电付费、零售电价、储能套利/未来放电价值没有显式进入 dispatch 私有利润代理。
3. 最近完整落盘 trace 覆盖到 episode 13，共记录到 14 个完整 episode；它显示 VPP 仍在买电。前三个 episode 平均吸收电量与最后三个 episode 平均吸收电量如下。
   - 前三集平均吸收电量：16.827158 MWh/episode
   - 最后三集平均吸收电量：27.969632 MWh/episode
   - 前三集平均 private_profit_proxy：-2464.757332
   - 最后三集平均 private_profit_proxy：-2885.887589
4. 如果吸收电量下降，说明策略在减少亏损性买电；如果没有明显下降，则不能说当前优化方向已经有效。下面趋势表给出逐 episode 证据。

### 4.1 DER 聚合口径补充说明

这一节补充说明“各个 VPP 到底聚合了什么 DER，以及这些 DER 在功率上占多大比重”。为了避免把不同物理含义混在一起，下面同时使用三种口径：

- 数量占比：某类 DER 的台数 / 当前场景全部 DER 台数。它回答“设备类型是不是很多”。
- 可调跨度占比：某类 DER 的平均可调功率跨度 / 全部 DER 的平均可调功率跨度。它回答“这类设备对动作空间有多大影响”。
- 吸收/注入能力占比：某类 DER 向负功率方向吸收用电、向正功率方向注入/减负荷的能力占比。它回答“它更像负荷资源还是发电/释放资源”。

项目内部的功率符号是：`P > 0` 表示 VPP 向配电网注入功率，或者减少等效负荷；`P < 0` 表示 VPP 从配电网吸收功率，也就是用电/充电/负荷增加。因此：

- EVCS、柔性负荷、HVAC 在当前模型中基本是吸收型或削减型负荷，不能像发电机一样主动向电网卖电。
- PV 和微型燃机是正向注入型资源，但 PV 的平均可用上限会被白天/夜间曲线稀释，不能只看装机容量。
- 储能既能充电也能放电，所以它的中点通常接近 0；但它对可调跨度非常重要。

- PV 有 `12` 个，占数量 `36.4%`，但平均注入能力占比为 `41.7%`，它确实是重要正向资源；只是 PV 夜间不可用，所以长期平均后无法完全抵消 EVCS、HVAC、柔性负荷。

- EVCS 数量只有 `2` 个，但吸收能力占比 `13.5%`，而注入能力为 `0.0%`，所以它会让对应 VPP 的可行域向负功率方向偏移。

## 9. 真实合理性判断
- 组合类型本身合理：校园、居民、EV hub、商业、工业、社区 VPP 中包含 PV、储能、柔性负荷、HVAC、EVCS、微型燃机，这符合配电网 VPP 的典型资源类型。
- 当前比例偏向灵活负荷压力测试：多个 VPP 含 EVCS/柔性负荷/HVAC，这些资源无法发电，只能减少或增加吸收功率，因此 FR 中点偏负是模型组合的自然后果。
- EVCS 当前是合成会话：EV 到达/离开时间由 EVCSModel 默认生成，而不是 ACN-Data 或真实充电订单数据。
- 电价当前主要是 synthetic/scarcity proxy 或 profile pack；代码中 data_sources 已把 OpenEI/CAISO/ACN 作为目标数据源，但 adapters 和 market settlement integration 仍标记缺失。
- HVAC 目前由默认 baseline 经 rated_power 裁剪得到初始负荷，这是可运行的简化模型，但不是经过建筑热模型/实测数据校准的真实 HVAC 数据集。

## 10. EV用户充电收入与储能未来放电收益是否存在
- dispatch 私有利润代理当前公式为：`private_profit_proxy = energy_market_revenue - der_operation_cost`。
- EV 用户充电收入：未发现独立收入项，例如 `charging_revenue/customer_payment/retail_tariff_revenue`。因此 EVCS 充电在能量市场口径下表现为 VPP 购电成本，而不是用户付费带来的收入。
- 储能未来放电收益：没有显式 terminal value、stored-energy value 或未来套利价值项。储能如果未来放电，未来 step 会产生正的 `energy_market_revenue`；但当前充电 step 没有收到“未来放电收益”的即时信用。
- portfolio reward 的 window_return 会汇总过去窗口的 `private_profit_proxy`，但它仍基于同一利润代理，不等价于储能跨时套利价值函数。

## 11. 建议
1. 如果论文主线强调 VPP 自利经济性，应新增 EV 用户充电收入/零售电价结算项，并区分批发购电成本、用户零售收入、灵活性服务收入。
2. 对储能加入显式 stored-energy terminal value 或面向未来价格的套利价值，否则充电动作在短期 reward 中天然偏负，credit assignment 很难。
3. 为 EVCS 接入真实或半真实会话数据：到达时间、离开时间、初始 SOC、目标 SOC、用户愿付价格/充电费率。
4. 对报告中的偏负 VPP，检查资源组合是否符合论文场景定位：如果是 EV/柔性负荷型 VPP，偏负合理；如果要表示发电型聚合商，应提高 PV/储能/燃机占比或加入可售电收益口径。
